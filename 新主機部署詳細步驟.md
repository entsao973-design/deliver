# 配送存證平台新主機部署詳細步驟

本文說明如何把「配送存證平台」部署到新的 Windows 主機，包含 Python 服務、SQL Server、Google Geocoding API、Google Cloud Vision OCR、IIS 反向代理，以及自動啟動方式。

請勿把正式密碼、Google API key、Google service account JSON 金鑰提交到 GitHub。

## 1. 建議部署架構

建議正式環境採用下列架構：

```text
手機 / 電腦瀏覽器
  -> https://dli.morris.com.tw
  -> IIS HTTPS / SSL / 反向代理
  -> http://127.0.0.1:8000 或 http://內部主機IP:8000
  -> Python 配送存證平台
  -> SQL Server Express / SQL Server
  -> storage/photos 檔案系統照片
  -> Google Geocoding API / Google Cloud Vision API
```

重點：

- 對外入口必須使用 HTTPS，PWA、手機定位、相機權限在手機上會比較穩定。
- Python 服務可以只聽內部網路或本機 `8000`，由 IIS 對外代理。
- SQL Server 儲存配送資料、帳號、達交狀態、照片路徑；照片檔案仍在 `storage/photos`。
- Google 金鑰只放在伺服器，不放在前端。

## 2. 新主機前置準備

### 2.1 Windows 與網路

1. 安裝 Windows 10 / 11 或 Windows Server。
2. 設定固定 IP，或至少確保 IIS 反向代理可以穩定連到 Python 主機。
3. 開放必要防火牆：
   - IIS 對外：TCP 443。
   - IIS 到 Python：TCP 8000，只允許內部來源即可。
   - Python 到 SQL Server：SQL Server TCP 連線埠，建議固定為 `1433` 或指定固定埠。
   - Python 對外到 Google：TCP 443。

### 2.2 建議資料夾

範例使用：

```text
C:\DeliveryProof\配送存證平台
C:\DeliveryProof\secrets
C:\DeliveryProof\backup
```

`C:\DeliveryProof\secrets` 用來放 Google Vision service account JSON。不要放在 `static`、`storage`、IIS 網站根目錄，避免被瀏覽器直接存取。

## 3. 安裝必要軟體

### 3.1 Python

安裝 Python 3.11 以上版本，安裝時建議勾選：

- Add Python to PATH
- Install launcher for all users

完成後用系統管理員 PowerShell 檢查：

```powershell
py --version
python --version
```

### 3.2 Git

安裝 Git for Windows，之後可用 GitHub 拉取程式：

```powershell
git --version
```

### 3.3 SQL Server ODBC Driver

安裝 Microsoft ODBC Driver 18 for SQL Server。若正式 `config.json` 仍設定 `ODBC Driver 17 for SQL Server`，則新主機也必須安裝 Driver 17，或把設定改成 Driver 18。

檢查已安裝驅動：

```powershell
Get-OdbcDriver -Name "*SQL Server*"
```

### 3.4 IIS、URL Rewrite、ARR

若使用 IIS 作 HTTPS 入口與反向代理，請安裝：

- IIS
- IIS URL Rewrite Module
- IIS Application Request Routing，簡稱 ARR

ARR 需在 IIS Manager 的伺服器層級啟用 Proxy。

## 6. SQL Server 設定

### 6.1 SQL Server 主機

1. 安裝 SQL Server Express / SQL Server。
2. 啟用 SQL Server Browser 或使用固定 TCP port。
3. 在 SQL Server Configuration Manager 啟用 TCP/IP。
4. 建議把 SQL Server TCP port 固定，例如 `1433`。
5. Windows 防火牆允許 Python 主機連入 SQL Server TCP port。

若使用 SQL Server Express named instance，正式部署仍建議改為固定埠，`config.json` 寫成：

```json
"server": "tcp:192.168.0.5,1433"
```

或同主機：

```json
"server": "tcp:127.0.0.1,1433"
```

### 6.2 建立資料庫與登入帳號

建議建立專用 SQL Login，例如：

```text
資料庫：DeliveryPhotoArchive
登入帳號：delivery_app
權限：DeliveryPhotoArchive db_owner
```

如果第一次啟動要讓系統自動建立資料庫與資料表，SQL Login 需要足夠權限。較穩定的正式做法是：

1. 先由管理者建立資料庫。
2. 第一次部署時暫時設 `initialize_schema=true`，讓系統補齊資料表/欄位。
3. 確認成功後改回 `initialize_schema=false`。

正式長期運作不建議每次啟動都自動改 schema。

## 7. 設定 config.json

若新主機沒有 `config.json`，先複製範本：

```powershell
Copy-Item config.example.json config.json
notepad config.json
```

正式環境建議重點如下，請把密碼改成正式值：

```json
{
  "storage_backend": "sqlserver",
  "database": {
    "type": "sqlserver",
    "server": "tcp:127.0.0.1,1433",
    "database": "DeliveryPhotoArchive",
    "username": "delivery_app",
    "password": "請填正式密碼",
    "driver": "ODBC Driver 18 for SQL Server",
    "encrypt": true,
    "trust_server_certificate": true,
    "create_database": false,
    "initialize_schema": false,
    "timeout": 5
  },
  "photo_root": "storage\\photos",
  "archive_root": "data\\archives",
  "upload_dir": "data\\uploads",
  "geocoding": {
    "enabled": true,
    "provider": "google",
    "api_key_env": "GOOGLE_GEOCODING_API_KEY",
    "country_hint": "Taiwan",
    "country_code": "TW",
    "timeout_seconds": 5
  },
  "scan_ocr": {
    "enabled": true,
    "provider": "google_vision",
    "credentials_file_env": "GOOGLE_APPLICATION_CREDENTIALS",
    "feature_type": "TEXT_DETECTION",
    "timeout_seconds": 10
  },
  "host": "0.0.0.0",
  "port": 8000
}
```

注意：

- `config.json` 不應提交 GitHub。
- 第一次建立新資料庫或升級欄位時，才把 `initialize_schema` 暫時改成 `true`。
- 成功啟動並確認資料表正常後，請改回 `false`。
- 若 ODBC 安裝的是 Driver 17，`driver` 就必須寫 `ODBC Driver 17 for SQL Server`。

## 8. Google Geocoding API 申請與設定

此功能用於 Excel 匯入後，依配送地址取得 GPS 經緯度，供「定位達交」比對 300 公尺內單據。

### 8.1 申請步驟

1. 進入 Google Cloud Console。
2. 建立或選擇一個 Google Cloud Project。
3. 啟用 Billing。
4. 啟用 Google Maps Platform 的 Geocoding API。
5. 建立 API key。
6. 限制 API key：
   - API restrictions：只允許 Geocoding API。
   - Application restrictions：若新主機對外 IP 固定，建議限制伺服器對外 IP。
   - 設定每日配額與預算警示。

### 8.2 寫入 Windows 環境變數

以系統管理員 PowerShell 執行：

```powershell
setx GOOGLE_GEOCODING_API_KEY "你的_Geocoding_API_KEY" /M
```

`/M` 代表寫入機器層級環境變數，重開機後仍會存在。已經執行中的 PowerShell、服務或排程不會自動讀到新值，請重新開 PowerShell，並重啟服務。

檢查是否存在時，不要輸出完整 key：

```powershell
$key = [Environment]::GetEnvironmentVariable("GOOGLE_GEOCODING_API_KEY", "Machine")
$key.Length
```

## 9. Google Cloud Vision API 申請與設定

此功能用於物流士端「掃號達交」OCR。手機只把取景框裁切後的小圖送到本平台後端，再由後端呼叫 Google Vision；Google 金鑰不會送到手機。

### 9.1 啟用 Vision API

1. 使用和 Geocoding 相同或不同的 Google Cloud Project 皆可。
2. 確認 Billing 已啟用。
3. 啟用 Cloud Vision API。
4. 建立 service account，例如：

```text
delivery-vision-ocr
```

5. 建立 service account key，格式選 JSON。
6. 下載 JSON 後放到新主機安全位置，例如：

```text
C:\DeliveryProof\secrets\delivery-vision-ocr.json
```

7. 設定檔案權限，只讓系統管理員與執行平台的服務帳號可讀取。

### 9.2 IAM 權限原則

目前系統是直接把圖片 bytes 傳給 Vision API，不使用 Google Cloud Storage 圖片路徑。因此通常不需要給 Storage 權限。

若未來改成把圖片先上傳 Cloud Storage，再把 Cloud Storage URI 交給 Vision，才需要另外授權 `Storage Object Viewer`。

### 9.3 寫入 Windows 環境變數

以系統管理員 PowerShell 執行：

```powershell
setx GOOGLE_APPLICATION_CREDENTIALS "C:\DeliveryProof\secrets\delivery-vision-ocr.json" /M
```

重新開 PowerShell 後檢查：

```powershell
$cred = [Environment]::GetEnvironmentVariable("GOOGLE_APPLICATION_CREDENTIALS", "Machine")
Test-Path $cred
```

檢查 Python 套件與 credentials 是否可載入：

```powershell
cd C:\DeliveryProof\配送存證平台
.\.venv\Scripts\python.exe -c "from google.cloud import vision; vision.ImageAnnotatorClient(); print('Vision client OK')"
```

如果這一步失敗，通常是：

- `GOOGLE_APPLICATION_CREDENTIALS` 路徑錯誤。
- JSON 檔案權限不足。
- Vision API 未啟用。
- 服務帳號金鑰被停用或刪除。

## 10. 第一次手動啟動與驗證

先不要急著做服務，先在 PowerShell 手動啟動：

```powershell
cd C:\DeliveryProof\配送存證平台
.\.venv\Scripts\python.exe server.py
```

另一個 PowerShell 測試：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json
```

瀏覽器測試：

```text
http://127.0.0.1:8000/driver
http://127.0.0.1:8000/admin
```

驗證項目：

- 登入頁可以開。
- 管理者可登入 `/admin`。
- 物流士可登入 `/driver`。
- 管理端可查詢配送資料。
- 可匯入 Excel。
- Excel 匯入後 geocode 狀態會逐步完成。
- 物流士端相機可拍照並上傳。
- 掃號達交可呼叫 OCR。

### 10.1 查詢 GPS 轉換狀態

可在 SQL Server Management Studio 查詢：

```sql
SELECT geocode_status, COUNT(*) AS count
FROM dbo.deliveries
GROUP BY geocode_status;
```

查看失敗原因：

```sql
SELECT TOP 50 delivery_date, company, invoice_no, address, geocode_status, geocode_error
FROM dbo.deliveries
WHERE geocode_status <> N'ok'
ORDER BY delivery_date DESC;
```

常見狀態：

- `ok`：已有經緯度。
- `pending`：等待轉換。
- `empty`：地址空白。
- `failed`：Google API 或地址解析失敗，請看 `geocode_error`。

## 11. 設定自動啟動

正式建議先使用 NSSM 建成 Windows Service；若不想加第三方工具，可用工作排程器。

### 11.1 先調整 start-server.ps1

目前 `start-server.ps1` 會優先找固定 Python 路徑，正式主機建議把 `$Python` 改成專案虛擬環境：

```powershell
$Python = Join-Path $ProjectRoot ".venv\Scripts\python.exe"
```

啟動腳本會讀取 `config.json` 的 port，若該 port 已經在本機啟動，會直接結束，避免重複開服務。

### 11.2 方法 A：NSSM Windows Service

1. 下載 NSSM 並放到固定位置，例如：

```text
C:\Tools\nssm\nssm.exe
```

2. 以系統管理員 PowerShell 建立服務：

```powershell
C:\Tools\nssm\nssm.exe install DeliveryProof "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" "-NoProfile -ExecutionPolicy Bypass -File ""C:\DeliveryProof\配送存證平台\start-server.ps1"""
```

3. 設定工作目錄與 log：

```powershell
C:\Tools\nssm\nssm.exe set DeliveryProof AppDirectory "C:\DeliveryProof\配送存證平台"
C:\Tools\nssm\nssm.exe set DeliveryProof AppStdout "C:\DeliveryProof\配送存證平台\server.out.log"
C:\Tools\nssm\nssm.exe set DeliveryProof AppStderr "C:\DeliveryProof\配送存證平台\server.err.log"
C:\Tools\nssm\nssm.exe set DeliveryProof Start SERVICE_AUTO_START
C:\Tools\nssm\nssm.exe set DeliveryProof AppRestartDelay 5000
```

4. 啟動服務：

```powershell
C:\Tools\nssm\nssm.exe start DeliveryProof
```

5. 檢查狀態：

```powershell
Get-Service DeliveryProof
```

6. 重新啟動：

```powershell
C:\Tools\nssm\nssm.exe restart DeliveryProof
```

建議建立專用 Windows 服務帳號執行，例如 `DeliverySvc`，並給它以下權限：

- 專案資料夾讀取。
- `storage/photos`、`data/uploads`、`data/archives`、log 檔寫入。
- `C:\DeliveryProof\secrets\delivery-vision-ocr.json` 讀取。
- 若 SQL Server 使用 Windows 驗證，還要給資料庫登入權限。

若 SQL Server 使用 SQL Login，服務帳號不需要 SQL Windows 登入，但仍需能讀取 `config.json`。

### 11.3 方法 B：Windows 工作排程器

若不想安裝 NSSM，可用工作排程器在系統啟動時執行。

以系統管理員 PowerShell 執行：

```powershell
schtasks /create /tn "DeliveryProof" /sc onstart /ru SYSTEM /rl HIGHEST /tr "powershell.exe -NoProfile -ExecutionPolicy Bypass -File ""C:\DeliveryProof\配送存證平台\start-server.ps1""" /f
```

立即啟動：

```powershell
schtasks /run /tn "DeliveryProof"
```

查詢：

```powershell
schtasks /query /tn "DeliveryProof" /v /fo list
```

注意：工作排程器可以自動啟動，但不像正式 Windows Service 那樣容易在 `services.msc` 裡管理。

## 12. IIS 反向代理設定

### 12.1 同主機部署

若 IIS 和 Python 在同一台：

```text
IIS HTTPS -> http://127.0.0.1:8000
```

IIS URL Rewrite 規則：

```text
Match URL: (.*)
Action type: Rewrite
Rewrite URL: http://127.0.0.1:8000/{R:1}
Append query string: true
```

### 12.2 IIS 與 Python 不同主機

若 IIS 在對外主機，Python 在內部主機：

```text
IIS HTTPS -> http://192.168.0.5:8000
```

Rewrite URL 改為：

```text
http://192.168.0.5:8000/{R:1}
```

### 12.3 SSL 與 PWA 注意事項

- 手機請使用 `https://dli.morris.com.tw` 進入，不要用內部 HTTP IP 安裝 PWA。
- Android PWA、相機、定位需要安全環境，正式入口用 HTTPS 是正確做法。
- Python 後端在 IIS 後面使用 HTTP 可以接受，因為瀏覽器看到的是 HTTPS。

## 13. 部署後驗證清單

### 13.1 基本網站

- `https://dli.morris.com.tw/driver` 可開物流士登入。
- `https://dli.morris.com.tw/admin` 可開管理登入。
- `https://dli.morris.com.tw/manifest.webmanifest` 可下載 manifest。
- Android Chrome 可加入主畫面並以 PWA 開啟。

### 13.2 登入與資料

- 管理者帳密可登入。
- 物流士帳密、車號可登入。
- 車號下拉與自行輸入都正常。
- 配送日期只顯示最近日期選項。
- 管理端查詢日期區間正常。

### 13.3 SQL Server

- 服務重開後仍可查詢資料。
- 匯入 Excel 後資料寫入 SQL Server。
- 照片路徑寫入資料庫，但圖片實體在 `storage/photos`。
- `initialize_schema` 正式運作時已改回 `false`。

### 13.4 Google Geocoding

- 匯入有地址的 Excel 後，`geocode_status` 會由 `pending` 轉為 `ok` 或 `failed`。
- `ok` 的資料有 `geocode_lat`、`geocode_lng`。
- 「定位達交」可取得手機定位，並比對 300 公尺內單據。

### 13.5 Google Vision OCR

- 物流士端「掃號達交」可開啟相機取景框。
- 拍攝後可看到 OCR 結果。
- 若查無單據，錯誤訊息會顯示 OCR 辨識內容供排查。
- 若找到一筆單據，可進入達交流程。
- 若找到多筆候選，可選擇單據。

### 13.6 離線與照片

- 物流士端斷網拍照會暫存。
- 網路恢復後會自動上傳。
- 管理端可檢視照片。
- 照片旋轉後檔案會被儲存。
- 封存照片 ZIP 可產生並下載。

## 14. 維護與備份

建議定期備份：

```text
SQL Server 資料庫 DeliveryPhotoArchive
storage/photos
data/archives
config.json
C:\DeliveryProof\secrets\delivery-vision-ocr.json
```

不建議只備份 SQL Server，因為照片實體不在資料庫內。

建議備份頻率：

- SQL Server：每日。
- `storage/photos`：每日或即時同步。
- `data/archives`：每日。
- `config.json` 與 Google JSON：每次變更後備份到安全位置。

## 15. 常見問題

### 15.1 網站打不開

檢查：

```powershell
Get-Service DeliveryProof
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json
Get-Content C:\DeliveryProof\配送存證平台\server.err.log -Tail 100
```

### 15.2 SQL Server 連不上

檢查：

- SQL Server 服務是否啟動。
- TCP/IP 是否啟用。
- SQL Server port 是否固定。
- Windows 防火牆是否允許。
- `config.json` 的 `server`、`database`、`driver` 是否正確。
- ODBC Driver 17/18 是否和設定一致。

### 15.3 Geocoding 顯示 API key not configured

檢查：

```powershell
[Environment]::GetEnvironmentVariable("GOOGLE_GEOCODING_API_KEY", "Machine")
```

如果剛設定完，請重啟 PowerShell 與平台服務。Windows 服務通常需要重啟後才讀到新的機器環境變數。

### 15.4 Vision OCR credentials 錯誤

檢查：

```powershell
$cred = [Environment]::GetEnvironmentVariable("GOOGLE_APPLICATION_CREDENTIALS", "Machine")
Test-Path $cred
```

並確認執行服務的 Windows 帳號有讀取 JSON 檔案的權限。

### 15.5 手機 PWA 看到舊畫面

可能是 Service Worker 或瀏覽器快取尚未更新。部署新版本後建議：

1. 確認 `static/app-version.json` 與 `static/service-worker.js` 版本已更新。
2. Android PWA 關閉後重新開啟。
3. 必要時移除主畫面捷徑後重新加入。
