# 封存日期自動載入設計

## 目標

管理者變更「封存日期」後，立即顯示該日期已存在的封存 ZIP，並預設全部勾選供下載。

## 設計

- 新增管理者 GET API，依 `delivery_date` 查詢 `archive_root` 中符合日期前綴的 ZIP。
- 查詢只讀取既有檔案，不重新封存或覆寫 ZIP。
- JSON 與 SQL Server repository 共用相同的檔案查詢函式。
- 前端監聽日期欄位的 `change` 事件並載入清單。
- 使用遞增請求序號，忽略較慢返回的舊日期結果。
- 封存完成後仍使用 API 回傳結果更新清單；每個項目預設勾選。
- 查無檔案時顯示「此日期尚無封存檔案」。

## 驗證

- repository 只列出指定日期的 ZIP，忽略其他日期及非 ZIP 檔。
- API 需要管理者權限並回傳指定日期封存清單。
- 前端日期變更會發出查詢，且舊回應不會覆蓋新日期結果。
- 清單中的所有封存檔預設勾選。
