# 配送紀錄維護功能設計

日期：2026-06-24

## 目標

在管理端「封存照片」旁新增「配送紀錄維護」頁籤，讓管理員依配送日期區間永久清除歷史資料。

日期區間包含開始日期與結束日期。清除範圍包含：

- 配送紀錄資料庫內的全部紀錄，不分未達交、已達交或已移入刪除區。
- `storage/photos` 下對應日期的已達交照片資料夾與照片。
- `data/archives` 下對應日期的已封存 ZIP。

此操作不可恢復。

## 管理介面

「配送紀錄維護」按鈕放在管理端頁籤列中，緊鄰「封存照片」。

維護頁面只提供本次需求必要的控制項：

- 開始日期。
- 結束日期。
- 紅色「永久清除」按鈕。

管理頁初始化時，開始日期與結束日期都預設為當天。

按下「永久清除」前先檢查：

- 兩個日期都必須填寫。
- 開始日期不得晚於結束日期。

通過檢查後顯示瀏覽器確認視窗，內容清楚列出日期區間與刪除項目，並包含「此清除無法恢復，請務必確定後執行」警告。只有按下確定才呼叫後端。

執行期間鎖定按鈕並顯示「清除中...」，避免重複送出。完成後顯示刪除摘要；失敗則顯示後端錯誤，不自動重試。

## 後端 API

新增管理員專用端點：

```text
POST /api/admin/maintenance/cleanup
```

請求內容：

```json
{
  "token": "管理員登入權杖",
  "start_date": "2026-01-01",
  "end_date": "2026-01-31"
}
```

後端再次驗證：

- 呼叫者必須是已登入的管理員。
- 日期必須是有效的 `YYYY-MM-DD`。
- 開始日期不得晚於結束日期。

成功回應提供可供畫面顯示的摘要：

```json
{
  "ok": true,
  "summary": {
    "deleted_records": 120,
    "deleted_photo_date_folders": 31,
    "deleted_archives": 58
  }
}
```

## 儲存層

JSON 與 SQL Server repository 都新增相同介面的日期區間清理方法。

### 資料庫紀錄

- JSON：在既有 repository lock 內，移除 `delivery_date` 落在閉區間內的所有紀錄，再以既有安全寫檔方式取代資料檔。
- SQL Server：使用參數化 `DELETE`，條件為 `delivery_date >= start_date AND delivery_date <= end_date`，並在交易成功後提交。
- 不檢查 `status` 或 `deleted_at`，因此未達交、已達交與刪除區資料都會清除。

### 照片

配送照片以 `YYYYMMDD` 日期資料夾保存。清理時巡覽 `photo_root` 的第一層，只刪除名稱可解析為日期且位於指定閉區間內的資料夾。

不根據資料庫回傳的 `photo_path` 逐筆刪除，避免漏掉已失去資料庫關聯的歷史照片。

### 封存 ZIP

封存檔依既有規則命名為 `YYYYMMDD_公司名稱.zip`。清理時只刪除：

- 位於 `archive_root` 第一層。
- 副檔名為 `.zip`。
- 檔名前八碼可解析為日期。
- 日期位於指定閉區間內。

其他檔案與不符合既有命名規則的 ZIP 不處理。

## 執行順序與失敗處理

資料庫與檔案系統無法共用同一個交易，因此採可安全重跑的順序：

1. 驗證日期。
2. 永久刪除資料庫紀錄。
3. 刪除日期照片資料夾。
4. 刪除日期封存 ZIP。
5. 回傳摘要。

若資料庫刪除失敗，不進行檔案清理。

若資料庫已成功但照片或 ZIP 清理失敗，API 回傳錯誤並指出清理未完整完成。管理員可用相同日期區間再次執行；即使資料庫紀錄已不存在，仍會重新清理日期資料夾與 ZIP，因此操作具冪等性。

不加入背景工作、復原區、排程、自動備份或額外的文字輸入確認，避免超出本次需求。

## 測試

### Repository 測試

JSON repository：

- 清除區間首日、區間中間與末日的全部紀錄。
- 保留區間外紀錄。
- 同時涵蓋未達交、已達交與刪除區紀錄。
- 刪除對應日期照片資料夾。
- 刪除對應日期且符合命名規則的 ZIP。
- 保留區間外照片、ZIP 與不符合命名規則的檔案。
- 相同區間重跑不報錯。
- 無效日期與反向日期區間被拒絕。

SQL Server repository：

- 以現有 SQL Server 測試替身或游標驗證參數化刪除條件與交易行為。
- 確認不以 `status` 或 `deleted_at` 篩選。

### Web 與前端測試

- 新頁籤位於「封存照片」旁。
- 維護頁包含開始日期、結束日期與「永久清除」按鈕。
- 前端送出前會檢查空白日期與反向區間。
- 確認視窗包含日期區間、刪除項目及不可恢復警告。
- 取消確認時不呼叫 API。
- 確認後呼叫正確端點與傳送正確內容。
- 清理期間使用既有按鈕鎖定工具。
- 後端拒絕非管理員、無效日期與反向區間。
- 成功回傳並顯示刪除摘要。

## 完成條件

- 管理員可從「封存照片」旁進入「配送紀錄維護」。
- 指定日期閉區間後，系統需要再次確認才會執行。
- 確認後，區間內所有配送紀錄、照片與封存 ZIP 都被永久清除。
- 區間外資料不受影響。
- JSON 與 SQL Server 後端行為一致。
- 相關自動化測試全部通過。
