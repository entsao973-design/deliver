# 配送存證平台維護指南

本文整理正式主機日常維護常用指令，包含工作排程啟動、停止、查詢、更新、log 檢查與常見排錯。

本文假設部署路徑為：

```text
C:\DeliveryProof\配送存證平台
```

若實際路徑不同，請自行替換。

## 1. 常用變數

開啟系統管理員 PowerShell，先設定：

```powershell
$ProjectRoot = "C:\DeliveryProof\配送存證平台"
$TaskName = "DeliveryProofServer"
$Port = 8000
```

切換到專案目錄：

```powershell
Set-Location $ProjectRoot
```

## 2. 檢查網站是否正在執行

### 2.1 檢查 port 8000

```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
```

若有結果，代表有程式正在 listen `8000`。

查是哪個行程：

```powershell
$Conn = Get-NetTCPConnection -LocalPort 8000 -State Listen
Get-Process -Id $Conn.OwningProcess
```

看命令列：

```powershell
Get-CimInstance Win32_Process -Filter "ProcessId=$($Conn.OwningProcess)" | Select-Object ProcessId,CommandLine
```

### 2.2 檢查平台 HTTP 回應

```powershell
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json -UseBasicParsing
Invoke-WebRequest http://127.0.0.1:8000/driver -UseBasicParsing
Invoke-WebRequest http://127.0.0.1:8000/admin -UseBasicParsing
```

若 Python 在另一台主機，請改用內部 IP：

```powershell
Invoke-WebRequest http://192.168.0.5:8000/static/app-version.json -UseBasicParsing
```

## 3. 工作排程管理

### 3.1 查詢工作排程是否存在

```powershell
Get-ScheduledTask -TaskName $TaskName
```

查執行狀態：

```powershell
Get-ScheduledTask -TaskName $TaskName | Get-ScheduledTaskInfo
```

### 3.2 啟動工作排程

```powershell
Start-ScheduledTask -TaskName $TaskName
```

啟動後檢查：

```powershell
Start-Sleep -Seconds 3
Get-NetTCPConnection -LocalPort 8000 -State Listen
Get-Content "$ProjectRoot\server.lifecycle.log" -Tail 20
```

### 3.3 停止工作排程

```powershell
Stop-ScheduledTask -TaskName $TaskName
```

再檢查 port：

```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
```

若工作排程停止後 Python 子行程仍在，找出 PID：

```powershell
$Conn = Get-NetTCPConnection -LocalPort 8000 -State Listen
Get-Process -Id $Conn.OwningProcess
```

確認是本專案 `server.py` 後再停止：

```powershell
Stop-Process -Id $Conn.OwningProcess
```

## 4. 建立或重建工作排程

若工作排程不存在，可用以下方式建立。

### 4.1 使用 SYSTEM 帳號

此方式適合正式主機。Google API key 若用系統環境變數 `/M` 設定，SYSTEM 也能讀取。

```powershell
$ProjectRoot = "C:\DeliveryProof\配送存證平台"
$TaskName = "DeliveryProofServer"

$Action = New-ScheduledTaskAction `
  -Execute "powershell.exe" `
  -Argument "-NoProfile -ExecutionPolicy Bypass -File `"$ProjectRoot\start-server.ps1`""

$Trigger = New-ScheduledTaskTrigger -AtStartup

$Settings = New-ScheduledTaskSettingsSet `
  -StartWhenAvailable `
  -RestartCount 3 `
  -RestartInterval (New-TimeSpan -Minutes 1)

Register-ScheduledTask `
  -TaskName $TaskName `
  -Action $Action `
  -Trigger $Trigger `
  -Settings $Settings `
  -User "SYSTEM" `
  -RunLevel Highest `
  -Description "配送存證平台 Python server"
```

建立後手動啟動：

```powershell
Start-ScheduledTask -TaskName $TaskName
```

### 4.2 使用目前登入帳號

若要用目前帳號執行：

```powershell
$User = "$env:USERDOMAIN\$env:USERNAME"

Register-ScheduledTask `
  -TaskName $TaskName `
  -Action $Action `
  -Trigger $Trigger `
  -Settings $Settings `
  -User $User `
  -RunLevel Highest `
  -Description "配送存證平台 Python server"
```

系統會要求輸入密碼。

注意：

- 若使用目前帳號，該帳號密碼變更後工作排程可能失效。
- 若 Google 環境變數只設在使用者層級，換成 SYSTEM 執行時讀不到；建議正式主機使用系統環境變數。

## 5. 更新系統版本

正式更新建議流程：

```powershell
$ProjectRoot = "C:\DeliveryProof\配送存證平台"
$TaskName = "DeliveryProofServer"

Set-Location $ProjectRoot

git status
Stop-ScheduledTask -TaskName $TaskName

git pull
.\.venv\Scripts\python.exe -m pip install -r requirements.txt

Start-ScheduledTask -TaskName $TaskName

Start-Sleep -Seconds 3
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json -UseBasicParsing
```

若 IIS 反向代理到同機：

```powershell
Invoke-WebRequest https://dli.morris.com.tw/static/app-version.json -UseBasicParsing
```

若 `git pull` 出現本機檔案衝突，先不要硬拉，先看：

```powershell
git status
git diff --stat
```

常見不應該在部署主機修改的檔案：

- 程式碼檔
- `requirements.txt`
- `static` 內前端檔

常見不應提交 GitHub 的檔案：

- `config.json`
- Google service account JSON
- log 檔
- `storage/photos`
- `.venv`

## 7. 前景模式測試

若工作排程啟動異常，可先用前景模式看錯誤。

先停止工作排程：

```powershell
Stop-ScheduledTask -TaskName $TaskName
```

確認 port 沒被佔用：

```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
```

前景啟動：

```powershell
Set-Location $ProjectRoot
.\.venv\Scripts\python.exe server.py
```

另開一個 PowerShell 測：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/driver -UseBasicParsing
```

測完後在前景視窗按 `Ctrl+C` 停止，再用工作排程啟動。

## 8. 常用 log

專案根目錄：

```text
server.lifecycle.log
server.out.log
server.err.log
server.import.log
server.import.worker.log
server.import.trace.log
```

查看：

```powershell
Get-Content "$ProjectRoot\server.lifecycle.log" -Tail 100
Get-Content "$ProjectRoot\server.err.log" -Tail 100
Get-Content "$ProjectRoot\server.out.log" -Tail 100
Get-Content "$ProjectRoot\server.import.log" -Tail 100
Get-Content "$ProjectRoot\server.import.worker.log" -Tail 100
```

用途：

- `server.lifecycle.log`：工作排程啟動、結束、執行帳號、Python 路徑。
- `server.err.log`：Python stderr。
- `server.out.log`：Python stdout。
- `server.import.log`：Excel 匯入流程主紀錄。
- `server.import.worker.log`：Excel 子行程解析紀錄。
- `server.import.trace.log`：Excel 匯入慢載入或堆疊診斷。

## 11. SQL Server 連線檢查


## 13. 備份建議

建議定期備份：

- SQL Server 資料庫。
- `storage/photos`。
- `config.json`。
- `C:\DeliveryProof\secrets`。
- win-acme renewal 設定：

```text
%ProgramData%\win-acme
```

簡易檔案備份範例：

```powershell
$Stamp = Get-Date -Format "yyyyMMdd-HHmmss"
$BackupRoot = "C:\DeliveryProof\backup\$Stamp"
New-Item -ItemType Directory -Force $BackupRoot

Copy-Item "$ProjectRoot\config.json" $BackupRoot -Force
Copy-Item "$ProjectRoot\storage" "$BackupRoot\storage" -Recurse -Force
Copy-Item "C:\DeliveryProof\secrets" "$BackupRoot\secrets" -Recurse -Force
```

SQL Server 請用 SQL Server Management Studio、維護計畫或 `BACKUP DATABASE` 做正式備份。

## 14. 常見日常情境

### 14.1 網站打不開

依序檢查：

```powershell
Get-Service W3SVC
Get-ScheduledTask -TaskName $TaskName | Get-ScheduledTaskInfo
Get-NetTCPConnection -LocalPort 8000 -State Listen
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json -UseBasicParsing
Get-Content "$ProjectRoot\server.lifecycle.log" -Tail 50
Get-Content "$ProjectRoot\server.err.log" -Tail 100
```

### 14.2 IIS 可以開，但平台 502

檢查 Python：

```powershell
Get-NetTCPConnection -LocalPort 8000 -State Listen
Invoke-WebRequest http://127.0.0.1:8000/driver -UseBasicParsing
```

若 Python 在別台：

```powershell
Test-NetConnection 192.168.0.5 -Port 8000
Invoke-WebRequest http://192.168.0.5:8000/driver -UseBasicParsing
```

### 14.3 匯入 Excel 失敗

查看：

```powershell
Get-Content "$ProjectRoot\server.import.log" -Tail 200
Get-Content "$ProjectRoot\server.import.worker.log" -Tail 200
Get-Content "$ProjectRoot\server.err.log" -Tail 100
```

若停在 Excel worker，檢查：

- `.venv` 套件是否完整。
- `openpyxl` 是否安裝。
- IIS request size 是否足夠。
- ARR timeout 是否足夠。

### 14.4 PWA 沒看到新版

檢查目前版本：

```powershell
Invoke-WebRequest https://dli.morris.com.tw/static/app-version.json -UseBasicParsing
```

手機端若仍看舊版：

- 關閉再開 PWA。
- 使用系統瀏覽器開一次網站。
- 等待 PWA 更新提示並按更新。
- 必要時移除主畫面 PWA 後重新安裝。

### 14.5 需要完整重啟

```powershell
Stop-ScheduledTask -TaskName $TaskName

$Conn = Get-NetTCPConnection -LocalPort 8000 -State Listen -ErrorAction SilentlyContinue
if ($Conn) {
  Stop-Process -Id $Conn.OwningProcess -Force
}

Start-ScheduledTask -TaskName $TaskName
Start-Sleep -Seconds 3
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json -UseBasicParsing
```
