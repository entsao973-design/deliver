# 配送存證平台部署主機安裝指南：工作排程版

本文說明如何在新的 Windows 部署主機安裝「配送存證平台」，並使用 Windows 工作排程器在開機時自動啟動網站。

Google Geocoding API 與 Google Cloud Vision API 的申請、金鑰、環境變數設定，請另外看 `Google服務申請與設定指南.md`。本文不放 Google 申請細節。

## 1. 部署架構

建議正式環境：

```text
使用者手機 / 電腦
  -> https://dli.morris.com.tw
  -> IIS HTTPS / SSL / 反向代理
  -> http://127.0.0.1:8000 或 http://內部主機IP:8000
  -> Python 配送存證平台
  -> SQL Server
  -> storage/photos 檔案系統照片
```

重點：

- Python 平台服務預設跑在 `8000`。
- 若 IIS 和 Python 在同一台主機，IIS 反向代理可指向 `http://127.0.0.1:8000`。
- 若 IIS 和 Python 不同主機，IIS 反向代理指向 Python 主機的內部 IP，例如 `http://192.168.0.5:8000`。
- 手機 PWA、相機、定位建議使用 HTTPS 網址進入。

## 2. 新主機前置準備

建議資料夾：

```text
C:\DeliveryProof\配送存證平台
C:\DeliveryProof\secrets
C:\DeliveryProof\backup
```

用途：

- `C:\DeliveryProof\配送存證平台`：程式。
- `C:\DeliveryProof\secrets`：Google Vision JSON 等機密檔，不能放進 Git。
- `C:\DeliveryProof\backup`：部署前備份或臨時備份。

建議防火牆：

- 對外網站由 IIS 開放 TCP `443`。
- Python 服務若只給 IIS 反向代理，可盡量限制 TCP `8000` 只允許內部或本機。
- SQL Server 使用固定 TCP port，例如 `1433` 或指定固定埠。
- Python 主機需要能連外 TCP `443`，供 Google API 使用。

## 3. 安裝必要軟體

### 3.1 Git

安裝 Git for Windows 後檢查：

```powershell
git --version
```

### 3.2 Python

安裝 Python 3.11 以上版本。安裝時建議勾選：

- Add Python to PATH
- Install launcher for all users

檢查：

```powershell
py --version
python --version
```

### 3.3 Microsoft ODBC Driver for SQL Server

安裝 Microsoft ODBC Driver 17 或 18 for SQL Server。

確認已安裝：

```powershell
Get-OdbcDriver -Name "*SQL Server*"
```

`config.json` 的 driver 名稱必須和實際安裝一致，例如：

```json
"driver": "ODBC Driver 18 for SQL Server"
```

或：

```json
"driver": "ODBC Driver 17 for SQL Server"
```

### 3.4 IIS、URL Rewrite、ARR

若要使用正式 HTTPS 網址，建議由 IIS 做 SSL 與反向代理。

需要安裝：

- IIS
- IIS URL Rewrite Module
- IIS Application Request Routing，簡稱 ARR

ARR 安裝後，要在 IIS Manager 的伺服器層級啟用 Proxy。

## 4. 取得程式碼

第一次部署才使用 `git clone`：

```powershell
cd C:\DeliveryProof
git clone https://github.com/entsao973-design/deliver.git 配送存證平台
cd C:\DeliveryProof\配送存證平台
```

如果資料夾已存在，以後更新請使用：

```powershell
cd C:\DeliveryProof\配送存證平台
git pull
```

不要在已有資料夾時重複 `git clone`，否則會出現：

```text
fatal: destination path '配送存證平台' already exists and is not an empty directory.
```

## 5. 建立 .venv 並安裝套件

部署主機需要建立自己的 `.venv`。`.venv` 不會提交 GitHub。

```powershell
cd C:\DeliveryProof\配送存證平台
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install --upgrade pip
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

確認主要套件：

```powershell
.\.venv\Scripts\python.exe -m pip show openpyxl
.\.venv\Scripts\python.exe -m pip show pyodbc
.\.venv\Scripts\python.exe -m pip show google-cloud-vision
```

注意：PowerShell 執行 `.venv` 裡的 Python 時，前面必須有 `.\`：

```powershell
.\.venv\Scripts\python.exe --version
```

不要寫成：

```powershell
.venv\Scripts\python.exe --version
```

## 6. 設定 config.json

第一次部署時複製範本：

```powershell
cd C:\DeliveryProof\配送存證平台
Copy-Item config.example.json config.json
notepad config.json
```

正式設定重點：

```json
{
  "storage_backend": "sqlserver",
  "database": {
    "type": "sqlserver",
    "server": "tcp:127.0.0.1,1433",
    "database": "DeliveryPhotoArchive",
    "username": "delivery_app",
    "password": "請填正式密碼",
    "driver": "ODBC Driver 18 for SQL Server",
    "encrypt": true,
    "trust_server_certificate": true,
    "create_database": false,
    "initialize_schema": false,
    "timeout": 5
  },
  "photo_root": "storage\\photos",
  "archive_root": "data\\archives",
  "upload_dir": "data\\uploads",
  "host": "0.0.0.0",
  "port": 8000
}
```

注意：

- `config.json` 不應提交 GitHub。
- 第一次建立資料庫或補欄位時，才把 `initialize_schema` 暫時改成 `true`。
- 確認資料庫與資料表正常後，正式運作請改回 `false`。
- 如果 SQL Server 不在本機，`server` 改成 SQL Server 主機 IP 與 port，例如 `tcp:192.168.0.5,1433`。

Google 相關設定請照 `Google服務申請與設定指南.md` 完成後，再把 `geocoding.enabled` 與 `scan_ocr.enabled` 改為 `true`。

## 7. SQL Server 準備

SQL Server 建議：

1. 啟用 TCP/IP。
2. 使用固定 TCP port。
3. 建立資料庫 `DeliveryPhotoArchive`。
4. 建立專用 SQL Login，例如 `delivery_app`。
5. 給該帳號 `DeliveryPhotoArchive` 的必要權限。

若要讓系統第一次啟動時自動建立資料庫與資料表，需要較高權限。正式環境較建議由資料庫管理者先建好資料庫，再讓平台補 schema。

驗證 SQL Server 網路連線：

```powershell
Test-NetConnection 127.0.0.1 -Port 1433
```

若 SQL Server 在其他主機：

```powershell
Test-NetConnection 192.168.0.5 -Port 1433
```

## 8. 第一次手動啟動測試

先不要建立工作排程，先手動測試平台能不能啟動。

```powershell
cd C:\DeliveryProof\配送存證平台
.\.venv\Scripts\python.exe server.py
```

另開一個 PowerShell 測試：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json
```

瀏覽器測試：

```text
http://127.0.0.1:8000/driver
http://127.0.0.1:8000/admin
```

手動測試完成後，在執行 `server.py` 的視窗按 `Ctrl + C` 停止。

## 9. 建立工作排程自動啟動

工作排程會執行：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File "C:\DeliveryProof\配送存證平台\start-server.ps1"
```

`start-server.ps1` 會優先使用：

```text
C:\DeliveryProof\配送存證平台\.venv\Scripts\python.exe
```

若 `.venv` Python 不存在，才會退回系統 `python`。

### 9.1 使用 schtasks 建立

以系統管理員 PowerShell 執行：

```powershell
schtasks /create /tn "DeliveryProof" /sc ONSTART /ru SYSTEM /rl HIGHEST /tr "powershell.exe -NoProfile -ExecutionPolicy Bypass -File ""C:\DeliveryProof\配送存證平台\start-server.ps1""" /f
```

參數意義：

- `/tn "DeliveryProof"`：工作名稱。
- `/sc ONSTART`：系統開機時啟動。
- `/ru SYSTEM`：使用 Local System 帳號執行。
- `/rl HIGHEST`：使用最高權限執行。
- `/tr ...`：實際執行的 PowerShell 腳本。
- `/f`：若同名工作已存在，直接覆蓋。

### 9.2 立即啟動工作

```powershell
schtasks /run /tn "DeliveryProof"
```

查詢工作狀態：

```powershell
schtasks /query /tn "DeliveryProof" /v /fo list
```

### 9.3 用工作排程器介面檢查

開啟：

```text
工作排程器
```

找到 `DeliveryProof`，檢查：

- General：
  - 使用者帳號：`SYSTEM` 或指定服務帳號。
  - 勾選 `Run with highest privileges`。
- Triggers：
  - `At startup`。
  - 可視情況設定延遲 30 秒，等 SQL Server 或網路完成啟動。
- Actions：
  - Program/script：`powershell.exe`
  - Arguments：`-NoProfile -ExecutionPolicy Bypass -File "C:\DeliveryProof\配送存證平台\start-server.ps1"`
  - Start in：`C:\DeliveryProof\配送存證平台`
- Settings：
  - 可勾選 `Run task as soon as possible after a scheduled start is missed`。
  - 可設定失敗後重新啟動。
  - 若工作已在執行，建議不要再啟動第二個實例。

## 10. 驗證工作排程啟動結果

啟動後檢查 port：

```powershell
netstat -ano | findstr ":8000"
```

檢查網站：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json
```

檢查 log：

```powershell
Get-Content C:\DeliveryProof\配送存證平台\server.err.log -Tail 100
Get-Content C:\DeliveryProof\配送存證平台\server.out.log -Tail 100
```

若 `server.err.log` 有 Python 套件缺少錯誤，重新安裝：

```powershell
cd C:\DeliveryProof\配送存證平台
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

## 11. 更新網站流程

更新不需要重開機。建議流程：

```powershell
cd C:\DeliveryProof\配送存證平台
schtasks /end /tn "DeliveryProof"
git pull
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
schtasks /run /tn "DeliveryProof"
```

確認版本：

```powershell
Invoke-WebRequest http://127.0.0.1:8000/static/app-version.json
```

若 `schtasks /end` 無法停止，可找出佔用 8000 的 PID：

```powershell
netstat -ano | findstr ":8000"
```

停止該 PID：

```powershell
Stop-Process -Id 看到的PID -Force
```

再啟動：

```powershell
schtasks /run /tn "DeliveryProof"
```

## 12. IIS 反向代理設定摘要

若 IIS 和 Python 在同一台：

```text
Rewrite URL: http://127.0.0.1:8000/{R:1}
```

若 Python 在內部另一台主機：

```text
Rewrite URL: http://192.168.0.5:8000/{R:1}
```

URL Rewrite 規則概念：

```text
Match URL: (.*)
Action type: Rewrite
Rewrite URL: http://127.0.0.1:8000/{R:1}
Append query string: true
```

手機請使用正式 HTTPS 網址安裝 PWA，例如：

```text
https://dli.morris.com.tw/driver
```

不要用內部 HTTP IP 安裝 PWA。

## 13. 備份建議

至少備份：

```text
SQL Server 資料庫 DeliveryPhotoArchive
storage/photos
data/archives
config.json
C:\DeliveryProof\secrets
```

## 14. 常見問題

### 14.1 git pull 被本地檔案擋住

若部署主機出現：

```text
Your local changes to the following files would be overwritten by merge
```

部署主機通常不應修改程式檔。若確定本地修改不要保留：

```powershell
git restore 檔名
git pull
```

舊版 Git 可用：

```powershell
git checkout -- 檔名
git pull
```

### 14.2 start-server.ps1 執行後沒反應

`start-server.ps1` 若偵測到 port `8000` 已經有服務在跑，會直接結束，避免開第二個服務。

檢查：

```powershell
netstat -ano | findstr ":8000"
```

### 14.3 .venv 裡缺套件

錯誤例：

```text
ModuleNotFoundError: No module named 'openpyxl'
```

處理：

```powershell
cd C:\DeliveryProof\配送存證平台
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

### 14.4 重新設定環境變數後沒有生效

若剛用 `setx ... /M` 設定機器環境變數，已經啟動的工作排程不會自動讀到新值。

請重啟工作：

```powershell
schtasks /end /tn "DeliveryProof"
schtasks /run /tn "DeliveryProof"
```

若仍不生效，再重新開 PowerShell 或重開機確認。
